Structure de navigation — Bottom tabs

Tu as dit : page 1 = promos/tendances (curiosité), page 2 = carte + recherche. Comment on organise les onglets ?

🔥
Discover
🗺️
Explorer
🔔
Alertes
👤
Profil
Discover = Promos + Tendances + Pour toi/Suivis
Explorer = Carte + Recherche produit précis
Alertes = Notifs promos des boutiques suivies
Profil = Favoris, tailles, paramètres

A — 4 onglets (Discover / Explorer / Alertes / Profil)

La recherche vit dans Explorer (carte). Alertes = onglet dédié pour les promos push. Pas d'onglet "Promos" séparé — les promos sont intégrées dans Discover.

🏠
Accueil
🔍
Recherche
🔥
Promos
👤
Profil
Accueil = Feed tendances + Pour toi/Suivis + carte mini
Recherche = Barre recherche + Carte + Filtres
Promos = Feed dédié promos (prix barrés, urgence)
Profil = Favoris, tailles, paramètres

B — 4 onglets (Accueil / Recherche / Promos / Profil)

Les promos ont leur propre onglet (comme TGTG). La recherche aussi. L'accueil est le feed de découverte pur. Carte intégrée dans Recherche.

🔥

🗺️

❤️
Favoris
👤
Profil
Feed = Promos 🔥 en haut + Tendances + Pour toi/Suivis
Carte = Carte plein écran + Recherche intégrée en haut
Favoris = Produits sauvegardés
Profil = Tailles, boutiques suivies, paramètres

C — 4 onglets (Feed / Carte / Favoris / Profil)

Comme ce qu'on a actuellement mais simplifié. Les promos sont en haut du feed (section héros). La carte est l'onglet de recherche. Favoris séparé pour accès rapide.

Ma recommandation : Option B. Promos en onglet dédié = plus de visibilité, raison de revenir. Recherche séparée du feed = chaque onglet a un job clair. C'est le pattern TGTG qui fonctionne très bien pour la rétention.
