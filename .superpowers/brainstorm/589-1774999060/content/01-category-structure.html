Comment structurer ~95 types de commerces ?

3 approches possibles — chacune avec des conséquences sur l'UX et la scalabilité

A

Liste plate élargie (15-20 catégories)

Comme aujourd'hui mais avec plus de choix : Mode, Chaussures, Bijoux, Beauté, Sport, Déco, Épicerie, + Livres, Jouets, Tech, Vins, Musique, Animaux, Maison, Enfants, Auto...

Avantages

  • Simple à implémenter
  • L'utilisateur voit tout d'un coup
  • Pas de navigation profonde

Inconvénients

  • 20 pills horizontaux → ça déborde, on scroll sans fin
  • Un magasin de running vs un magasin de golf = tous les deux "Sport"
  • Ne scale pas au-delà de 20
  • Le marchand doit choisir UNE catégorie (ou on devine)
B

Arbre hiérarchique 2 niveaux (comme Vinted)

Niveau 1 : 10-12 grandes catégories (Mode, Sport, Tech, Maison...)
Niveau 2 : Sous-catégories au clic (Mode → Femme / Homme / Enfant / Chaussures / Accessoires)

Avantages

  • Scale à l'infini — on ajoute des sous-catégories sans changer l'UI
  • Familier (Vinted, Amazon, Leboncoin)
  • Permet des filtres précis ("Sport > Running" vs "Sport > Golf")
  • Le marchand peut être classé plus finement

Inconvénients

  • Plus complexe côté UI (dropdown, drawer, ou page dédiée)
  • L'auto-catégorisation depuis le POS est plus dure (2 niveaux à deviner)
  • Risque d'arbre vide au début (sous-catégories sans produits)
C

Tags multiples + catégorie principale (hybride)

1 catégorie principale par marchand (parmi 12-15) + tags libres sur les produits (sneakers, vintage, bio, running, vegan...).
L'utilisateur filtre par catégorie ET/OU par tags. Les tags se remplissent progressivement.

Avantages

  • Flexible — un produit peut avoir plusieurs tags
  • Scale naturellement (nouveaux tags = 0 changement UI)
  • Les tags populaires remontent automatiquement
  • Le POS donne le nom → on extrait des tags ("Air Force" → sneakers, Nike)

Inconvénients

  • Moins structuré — le consommateur ne peut pas "browser par rayon"
  • Les tags doivent être normalisés (sinon chaos : "chaussure" vs "shoes" vs "sneakers")
  • Demande un système d'extraction/suggestion de tags

💡 Ma recommandation : B + C hybride

Catégorie principale hiérarchique (2 niveaux, assignée au marchand) pour le browsing structuré + Tags extraits automatiquement des noms de produits POS pour le filtrage fin.

Exemple : Marchand "Sneakers District" → Catégorie Mode > Chaussures + ses produits ont les tags auto-extraits sneakers, Nike, New Balance, running.
Le consommateur peut browser "Mode > Chaussures" OU chercher le tag "Nike".
