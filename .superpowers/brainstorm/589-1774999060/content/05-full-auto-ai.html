Full auto par IA — Le marchand ne choisit rien

Le POS se connecte, les produits tombent, l'IA catégorise tout. Zéro intervention.

Le pipeline en 4 étapes

1

POS sync — Les produits arrivent

200 produits avec nom, prix, EAN, photo. Déjà implémenté.

2

EAN lookup — La marque et la catégorie standard

UPCitemdb donne "Nike", "Air Force 1", catégorie "Footwear". Déjà implémenté.

3

🆕 IA batch — Catégorisation intelligente

On envoie les 200 noms de produits en UN appel à Claude Haiku. L'IA retourne pour chaque produit :

Entrée : "AIR FORCE 1 07 BLC 42"
→ catégorie: Chaussures
→ sous-catégorie: Sneakers
→ marque: Nike
→ genre: Mixte
→ couleur: Blanc
→ taille: 42
→ tags: [sneakers, nike, blanc, air-force]
Entrée : "Bougie Diptyque Baies 190g"
→ catégorie: Maison & Déco
→ sous-catégorie: Bougies
→ marque: Diptyque
→ tags: [bougie, parfum-intérieur, diptyque]
4

🆕 Catégorie marchand auto-déduite

On regarde la catégorie la plus fréquente parmi ses produits → c'est sa catégorie principale. Si 80% des produits sont "Chaussures", le marchand est classé "Chaussures".

Sneakers District : 180 chaussures, 15 accessoires, 5 vêtements
→ catégorie marchand : Chaussures (90%)
→ catégories secondaires : Bijoux & Accessoires, Mode

🛡️ Filet de sécurité

✓

Confiance score — L'IA retourne un score de confiance. Si < 70%, le produit est marqué "Non catégorisé" → le marchand peut corriger dans le dashboard.

✓

Correction marchand — Une page "Vérifier vos catégories" dans le dashboard. Le marchand voit les catégories auto-assignées et peut corriger d'un tap. Mais il n'est pas obligé.

✓

Fallback règles — Si l'IA est indisponible, un système de mots-clés basique fait le travail (moins précis mais suffisant).

✓

Apprentissage — Chaque correction du marchand améliore le système pour les marchands suivants (mots-clés → catégorie confirmée).
