Quelle précision attendre de l'IA ?

Estimation honnête basée sur la qualité variable des noms POS

Tout dépend de ce que le POS nous donne

😎 Nom clair et complet ~95-98%
"Nike Air Force 1 '07 Blanc Homme Taille 42"

→ Catégorie, sous-catégorie, marque, couleur, genre, taille — tout est dans le nom. L'IA ne peut presque pas se tromper.

🤔 Nom abrégé mais lisible ~80-90%
"AF1 07 BLC 42"  •  "Robe lin nat M"  •  "NB 574 GR"

→ L'IA comprend "AF1" = Air Force 1, "NB" = New Balance, "BLC" = blanc. Les LLM sont très forts sur les abréviations commerciales. La catégorie (Chaussures) est évidente, la sous-catégorie (Sneakers) aussi.

😬 Code interne / cryptique ~40-60%
"REF-2847-BLK"  •  "ART.5923"  •  "PROD-001"

→ Aucune info exploitable dans le nom seul. MAIS : si on a l'EAN, on retrouve tout via la base produit. Si on a la photo, on peut aussi déduire. Sans rien → "Non catégorisé".

Ce qui booste la précision au-delà du nom seul

📊 EAN / Code-barres

+15-20%

L'EAN donne la marque exacte et souvent la catégorie via UPCitemdb. Transforme un "REF-2847" en "Nike Air Max 90".

🏪 Contexte boutique

+5-10%

Si 180/200 produits sont des chaussures, le produit ambigu "ART.5923" est probablement une chaussure aussi.

📸 Photo produit

+10-15%

Vision IA sur la photo → "c'est une sneaker blanche". Possible mais plus cher (modèle vision). Phase 2 du pipeline.

💰 Prix

+5%

Un produit à 3€ dans une parfumerie n'est pas un parfum (probablement un accessoire). Le prix aide à discriminer.

💡 Pour mettre en perspective

90% de précision sur la catégorie = 180 produits bien classés sur 200.
Les 20 restants seront soit "Non catégorisé" (si confiance basse), soit mal classés.

Le marchand peut corriger en 2 minutes dans le dashboard. Mais même sans correction, 90% c'est largement suffisant pour que le consommateur trouve les produits via la recherche et les filtres.

Alternative sans IA : le marchand catégorise 200 produits à la main = 30-45 minutes de travail ennuyeux = il ne le fera jamais.

Conclusion : 90% automatique + correction optionnelle > 0% parce que le marchand n'a pas le temps.
