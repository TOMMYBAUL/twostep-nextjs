Comment le consommateur utilise les tags ?

Les tags (marque, couleur, style...) sont extraits automatiquement des produits POS. Comment les rendre utiles au consommateur ?

A

Filtres dans le panneau "Filtres" (style Vinted)

Les tags alimentent les filtres contextuels du panneau de filtres. Le consommateur tape sur l'icône filtre → panneau slide → il voit Marque, Couleur, Genre, etc. avec les valeurs disponibles.

Avantages

  • Pattern connu et éprouvé (Vinted, Zalando, ASOS)
  • Tout est groupé au même endroit
  • Nombre de résultats visible avant de valider

Inconvénients

  • Un tap de plus pour y accéder (icône filtre → panneau)
  • L'utilisateur doit savoir qu'il peut filtrer par marque
B

Recherche intelligente (les tags nourrissent l'autocomplete)

Le consommateur tape "Nike blanc" dans la barre de recherche → l'autocomplete propose des résultats basés sur les tags : produits Nike de couleur blanche disponibles à proximité.

Avantages

  • Ultra naturel — on tape ce qu'on cherche
  • Combine marque + couleur + taille en une seule requête
  • Résultats mixtes : produits ET boutiques

Inconvénients

  • L'utilisateur doit savoir quoi chercher (pas de découverte)
  • Nécessite un moteur de recherche plus avancé

💡 Ma recommandation : Les deux (A + B)

Ce n'est pas un choix exclusif. Les tags alimentent à la fois :

1. Le panneau de filtres (quand on browse une catégorie → on affine par marque/couleur/prix)
2. La recherche intelligente (quand on sait ce qu'on cherche → "nike blanc 42")

C'est exactement ce que fait Vinted : tu peux filtrer par marque dans les filtres, OU taper "Nike" dans la recherche. Les deux mènent aux mêmes résultats.
