Contourner les 30% Apple — Stratégie stories mobile

Comment laisser les marchands faire des stories sur téléphone sans payer la taxe Apple

✅ Aujourd'hui : pas de problème

Two-Step est une web app / PWA. Pas d'App Store, pas de taxe Apple. Le marchand paie via Stripe sur le site web. Les 39€/mois arrivent à 100% chez nous (moins les ~2.9% Stripe).

⚠️ Le problème futur : si app native iOS

Si on publie une app sur l'App Store avec un abonnement marchand dedans → Apple prend 30% (soit ~12€/mois sur 39€). Sur 50 marchands = 600€/mois perdus.

3 stratégies possibles

A

Rester en PWA (recommandé pour le lancement)

Pas d'app native. Le consommateur "installe" Two-Step depuis Safari (Ajouter à l'écran d'accueil). Le marchand utilise le dashboard web sur PC et le même site sur son téléphone pour les stories.

✅ 0% Apple · ✅ Stories sur mobile (navigateur) · ✅ Pas de validation App Store · ✅ Déploiement instantané
❌ Pas de push notifications natives (mais les web push fonctionnent) · ❌ Pas dans l'App Store (découvrabilité)
B

App native consumer (gratuite) + Dashboard web (payant)

L'app iOS est uniquement pour les consommateurs (gratuite, pas de paiement). Les marchands n'ont aucune fonction payante dans l'app. Ils s'abonnent et gèrent tout sur le site web.

✅ 0% Apple (pas de paiement dans l'app) · ✅ App Store pour les consommateurs · ✅ Push natives
❌ Les marchands doivent ouvrir le navigateur pour les stories sur mobile · ⚠️ Modèle "Netflix" accepté par Apple
Le bouton "Ajouter" (story) dans l'app consumer apparaît uniquement si le user est marchand → redirige vers le dashboard web dans le navigateur. Pas d'in-app purchase.
C

App native avec abonnement hors-app

L'app a les features marchands mais aucun bouton d'achat. Le marchand doit s'être abonné sur le site web avant. L'app vérifie l'abonnement et débloque les features. Comme Spotify/Netflix.

✅ 0% Apple · ✅ Stories nativement dans l'app · ✅ Meilleure UX marchand
⚠️ Apple peut rejeter si mal exécuté · ❌ Plus complexe à implémenter · ❌ Depuis mars 2024, Apple autorise les liens externes mais prend quand même 27%

Ma recommandation : A maintenant → B plus tard

Phase 1 (maintenant) : Rester en PWA. Le dashboard marchand est responsive — les stories se créent sur le téléphone via le navigateur. Le marchand prend la photo, écrit le texte, planifie. Tout marche. Zéro Apple.

Phase 2 (quand on a 1000+ consommateurs) : Publier une app native iOS/Android pour les consommateurs uniquement (gratuite). Les marchands continuent de gérer tout sur le web. Si un marchand ouvre l'app consumer et qu'il est identifié comme marchand, le bouton "+" sur les stories le redirige vers son dashboard web.

Résultat : Apple ne voit qu'une app gratuite. Le paiement reste 100% web. Les stories marchands passent par le navigateur mobile (le site est déjà mobile-first).
